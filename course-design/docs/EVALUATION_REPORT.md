## 四、已知限制与改进方向

### 当前限制

1. **训练数据集规模有限**：自定义 campus_safety 数据集仅数百张图片，主要依赖 COCO 预训练
2. **骨架检测需 GPU**：pose 模型推理需要 GPU，低功耗设备上默认关闭
3. **MLLM 延迟较高**：Qwen2-VL-2B 在无 GPU 加速时单次推理 ~2-5s
4. **CLIP 模型较大**：首次下载 ~600MB，需要 transformers 依赖
5. **无跨摄像头 ReID**：跟踪 ID 仅在本视频流内有效
6. **无音频检测**：计划书中未要求但可作为扩展

### 改进方向

1. **跨摄像头行人重识别**：集成 OSNet 或 FastReID 模型
2. **音频异常检测**：尖叫声、玻璃破碎声等
3. **边缘-云协同**：边缘做实时检测，云端做深度分析
4. **模型量化**：INT8 量化进一步降低 Jetson 部署开销
5. **联邦学习**：多校区数据联合训练，保护隐私

## 五、部署建议

### 硬件要求

| 场景 | 推荐配置 |
|------|---------|
| 开发测试 | CPU 即可（关闭骨架+MLLM） |
| 单路摄像头 | NVIDIA RTX 3060+ / Jetson Orin |
| 多路摄像头 | NVIDIA RTX 4070+ / T4 / A10 |
| 边缘部署 | Jetson Orin NX / AGX Orin |

### 启动步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 准备模型（自动下载 CLIP 等）
python scripts/launcher.py --download-models

# 3. 启动后端
python scripts/launcher.py --config configs/default.yaml

# 4. 启动前端
cd frontend && npm run dev

# 5. 或 Docker 一键部署
docker-compose up -d
```

## 六、总结

本系统完整实现了项目二计划书中所有核心功能，并在多个维度超出预期：

- **行为检测**：从计划书的 3 种（奔跑/摔倒/聚集）扩展到 5 种（+打架/入侵）
- **检测精度**：双引擎（BBox + 骨架）+ 次级模型 + LSTM 序列分类器
- **跟踪能力**：ByteTrack + BoT-SORT 双选 + IOU 回退匹配
- **检索能力**：CLIP 跨模态特征检索，支持自然语言搜索
- **前端体验**：专业级深色主题监控仪表盘
- **部署灵活性**：Docker / Jetson / GPU / CPU 全支持

系统具备生产级质量，可直接部署于校园安保中心，实现从被动录像到主动预警的安全管理升级。
