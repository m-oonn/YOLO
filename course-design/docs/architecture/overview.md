## 模块职责

### 1. 前端层 (frontend/)

| 模块 | 职责 | 关键文件 |
|------|------|----------|
| API Client | 封装所有 HTTP/WebSocket 请求 | `src/api/client.js` |
| Stores | 状态管理（检测、事件） | `src/stores/` |
| Views | 页面视图 | `src/views/*.vue` |
| Router | 路由配置 | `src/router/index.js` |

### 2. 后端层 (backend/)

| 模块 | 职责 | 关键文件 |
|------|------|----------|
| API 层 | HTTP 端点定义 | `api/*.py` |
| 服务层 | 业务逻辑封装 | `detection_manager.py` |
| 安全层 | 认证、限流、验证 | `security.py`, `limiter.py` |
| 单例管理 | 全局实例共享 | `alarm_singleton.py`, `store.py` |

### 3. 核心引擎层 (core/)

| 模块 | 职责 | 关键文件 |
|------|------|----------|
| Pipeline | 检测流水线编排 | `pipeline.py` |
| Skeleton | 17 点骨架提取 | `skeleton.py` |
| Behavior | 行为分析（跑/跌倒/打架/拥挤） | `behavior_analyzer.py` |
| Rules | 规则引擎（bbox 行为检测） | `rules.py` |
| Alarm | 告警通知系统 | `alarm_engine.py`, `notifiers.py` |
| MLLM | 场景理解增强 | `mllm/mllm_sidecar.py` |
| Config | 配置管理 | `config.py` |

## 核心数据流

```
视频源 (摄像头/文件)
    │
    ▼
┌─────────────────┐
│  DetectionPipeline │
│  (读取帧 → YOLO 推理) │
└────────┬──────────┘
         │
    ┌────▼─────────────────────────────┐
    │         后处理                    │
    │  1. NMS 过滤                      │
    │  2. 目标跟踪 (ByteTrack)           │
    │  3. 骨架提取 (17 keypoints)        │
    └────┬─────────────────────────────┘
         │
    ┌────▼─────────────────────────────┐
    │         行为分析                  │
    │  ┌────────────────────────────┐  │
    │  │ RulesEngine (bbox 规则)     │  │
    │  │ - 跌倒 (宽高比 + 过渡)       │  │
    │  │ - 拥挤 (ROI 内人数)         │  │
    │  │ - 打架 (交互分析)           │  │
    │  └────────────────────────────┘  │
    │  ┌────────────────────────────┐  │
    │  │ BehaviorAnalyzer (骨架规则)  │  │
    │  │ - 跌倒 (角度 + 速度)         │  │
    │  │ - 奔跑 (速度 + 步态)         │  │
    │  └────────────────────────────┘  │
    └────┬─────────────────────────────┘
         │
    ┌────▼─────────────────────────────┐
    │         事件处理                  │
    │  1. 生成 Event 对象              │
    │  2. 存储到 SQLite                │
    │  3. 告警引擎处理                 │
    │  4. 推送到 WebSocket 客户端      │
    │  5. MLLM 场景理解 (可选)          │
    └──────────────────────────────────┘
```

## 关键设计模式

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| 单例模式 | `alarm_singleton.py`, `store.py` | 全局唯一实例 |
| 策略模式 | `RulesEngine`, `BehaviorAnalyzer` | 可插拔规则引擎 |
| 观察者模式 | `DetectionPipeline` + WebSocket | 事件发布订阅 |
| 工厂模式 | `AlarmStore`, `EventsStore` | 对象创建封装 |
| 装饰器模式 | `@limiter.limit()`, FastAPI 中间件 | 横切关注点 |
