## 二、核心技术栈对比

### 2.1 目标检测

| 计划书要求 | 实际实现 | 状态 | 说明 |
|-----------|---------|------|------|
| YOLOv11 | `ultralytics>=8.4.0`，支持 `yolov11x.pt` | 已实现 | 主检测模型 |
| YOLOv12 | 支持 `yolo12s.pt` 等 | 已实现 | 可通过配置切换 |
| YOLOv26 | **未实现** | 未实现 | 计划书中提及但代码中无支持 |
| mAP@50 达 0.952 | 依赖 Ultralytics 框架性能 | 间接实现 | 使用官方预训练权重 |
| 推理速度 30ms | `inference_scale` 可调，`GPUManager` 优化 | 已实现 | 支持推理缩放+FP16+GPU 预处理 |

**分析**：
- YOLOv11 和 YOLOv12 已完整支持，通过 `model_path` 配置即可切换
- YOLOv26 在计划书中提及但在实际代码中未见支持（可能为笔误或未来扩展）
- 性能优化方面实现了超出计划书的策略：半精度推理、动态质量调整、CUDA 缓存清理

### 2.2 目标跟踪

| ByteTrack | `tracker: "bytetrack"`（默认） | 已实现 |

### 2.3 行为分析

| 计划书要求 | 实际实现 | 状态 | 说明 |
|-----------|---------|------|------|
| 基于骨架的动作识别 | `SkeletonExtractor` + `BehaviorAnalyzer` | 已实现 | COCO 17 关键点 |
| 奔跑检测 | `RulesEngine._detect_running()` + `SkeletonRunningRule` | 已实现 | BBox + Skeleton 双路检测 |
| 摔倒检测 | `RulesEngine._detect_fall()` + `SkeletonFallRule` | 已实现 | 多信号融合 |
| 聚集检测 | `RulesEngine._detect_crowd()` + `CrowdDensityAnalyzer` | 已实现 | DFS 聚类 + Voronoi 密度 |
| 打架检测 | `RulesEngine._detect_fight()` + `SkeletonFightRule` | 已实现 | 多因子评分 + 时序分析 |
| 禁区入侵 | `RulesEngine._detect_intrusion()` | 已实现 | 多边形区域检测 |
| 车辆入侵 | `RulesEngine._detect_vehicle_intrusion()` | 已实现 | 超出计划书要求 |

**分析**：
- 计划书要求的 4 种异常行为（奔跑、摔倒、聚集、打架）全部实现
- 额外实现了禁区入侵和车辆入侵检测，功能超出计划
- 采用 BBox 规则引擎 + 骨架分析器双路架构，检测精度高于计划书预期

### 2.4 多模态理解（MLLM）

## 五、架构设计对比

### 5.1 计划书隐含架构 vs 实际架构

**计划书隐含架构**（基于功能描述推断）：
```
摄像头 → YOLO检测 → 跟踪 → 行为分析 → 告警 → 存储
                ↓
            MLLM场景描述
                ↓
            Web展示
```

**实际架构**（更复杂和完善）：
```
摄像头/视频文件/RTSP
    ↓
FrameCapture (独立线程)
    ↓
DetectionThread (独立线程)
    ↓
┌─────────────┬─────────────┬─────────────┐
↓             ↓             ↓             ↓
YOLO推理    RulesEngine   Skeleton      Secondary
+跟踪       (BBox规则)    Analysis      Models
            │             (Pose)        (CPU)
            ↓             ↓
        EventMerge ←─── 骨架事件
            ↓
        EventsStore (SQLite)
            ↓
    ┌───────┼───────┐
    ↓       ↓       ↓
AlarmEngine  ClipRecorder  MLLMSidecar
(分级/抑制)  (视频存档)    (场景描述)
    ↓                       ↓
REST API ←─────────────── WebSocket
    ↓
Vue.js Frontend
```

**分析**：
- 实际架构比计划书更为复杂和完善
- 引入了多线程架构（捕获/检测分离）、双路行为检测（BBox+Skeleton）、异步 MLLM Sidecar 等高级设计
- 数据持久化、告警管理、视频存档等功能超出计划书要求
