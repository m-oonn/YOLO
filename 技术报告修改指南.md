### P0-2：新增"双引擎行为分析架构"章节

**当前问题**：报告仅用一句话描述"滑动窗口+规则引擎"，完全缺失骨架分析引擎

**修改方案**：在第三章新增3.2节，内容如下：

```markdown
### 3.2 双引擎行为分析架构

系统采用 BBox规则引擎 + 骨架分析引擎 双引擎互补架构：

#### BBox 规则引擎（core/rules.py）
基于目标检测边界框的几何特征进行行为判别：

| 检测类型 | 核心算法 | 关键参数 |
|----------|----------|----------|
| 奔跑检测 | 像素速度追踪 + 透视校正（px/s → km/h） | speed_px_s: 350, min_duration_s: 0.4s |
| 摔倒检测 | 宽高比变化 + 连续帧确认 + 快速过渡检测 | upright_aspect_min: 1.1, confirm_frames: 6 |
| 人群聚集 | 空间聚类（DFS连通分量）+ 透视补偿 | min_people: 4, proximity_px: 180 |
| 入侵检测 | 多边形区域 + BBox四角点+中心点判别 | zones: [] |
| 打架检测 | 多因子评分（距离/IoU/接近度/混乱度/移动量） | required_score: 4 |
| 车辆入侵 | 车辆类别识别（5类）+ 区域检测 | zones: [] |

#### 骨架分析引擎（core/behavior_analyzer.py）
基于17点COCO人体关键点的多信号融合分析：

| 检测类型 | 核心算法 |
|----------|----------|
| 骨架奔跑 | 线性回归速度 + 步态分析 + 自适应阈值 |
| 骨架摔倒 | 多信号融合（躯干角度40°阈值 / 头部速度 / 宽高比 / 髋部位移 / 角度变化率） |
| 骨架打架 | 时序分析 + 肢体交互频率 + 腕部速度（1.5m/s阈值） |
| 人群密度 | Voronoi图 / 凸包面积 / 社交密度 三指标融合 |
| 骨架入侵 | 髋中点（hip-center）区域多边形判别 |

#### 事件去重
骨架事件（标注 detection_method=skeleton）优先于 BBox 事件，
按 (event_type, track_id) 去重，骨架误报率更低。
```

**依据**：`core/rules.py`、`core/behavior_analyzer.py`、`core/skeleton.py`、`core/constants.py` 骨架定义

### P0-4：新增"TensorRT 推理加速"内容

```markdown
### 5.3 TensorRT 推理加速

#### 转换流水线
YOLO .pt模型 → ONNX（export_yolo_to_onnx）→ TensorRT引擎（build_tensorrt_engine）
→ TRTModelWrapper（Ultralytics兼容包装）

#### 精度模式

| 模式 | 推理延迟 | 显存占用 | 精度损失 | 适用场景 |
|------|----------|----------|----------|----------|
| FP32 | ~30ms | 高 | 基准 | 最高精度需求 |
| FP16 | ~15ms | 中（~1.2GB） | <1% | 速度/精度平衡（推荐） |
| INT8 | ~10ms | 低 | ~2-3% | 边缘设备/高吞吐 |

#### 特性
- 自动回退：TensorRT不可用时自动回退到PyTorch推理
- DLA加速：Jetson平台支持Deep Learning Accelerator
- 动态尺寸：支持多分辨率输入
- 引擎缓存：首次构建后持久化，后续启动秒级加载
```

**依据**：`core/tensorrt_utils.py`（~700行）、`core/pipeline.py` 第144-156行、`scripts/convert_to_tensorrt.py`

### P0-6：告警系统扩展

**当前问题**：仅描述为"弹窗+日志+WebSocket三通道"

**修改方案**：将3.3节（告警引擎模块）替换为：

```markdown
### 3.5 告警引擎（core/alarm_engine.py）

#### 三级告警分类
| 级别 | 标签 | 触发事件 |
|------|------|----------|
| CRITICAL（严重） | 红色 | fight, fall |
| WARNING（警告） | 橙色 | intrusion, crowd, vehicle_intrusion |
| INFO（提示） | 蓝色 | running |

#### 告警生命周期管理
1. **抑制**（Suppression）：同类型事件30s冷却窗口，避免重复告警
2. **聚合**（Aggregation）：60s窗口内同类告警合并计数
3. **自动升级**（Escalation）：300s内未处理的严重告警自动升级
4. **确认/解决**：API支持 acknowledge（确认）和 resolve（解决）操作

#### 通知通道
- 系统内弹窗（前端实时卡片）
- 结构化日志（数据库持久化）
- WebSocket 实时推送（多终端同步）
- 预留 Webhook/Email/SMS 接口
```
